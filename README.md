#Proyecto Extra – Proyecto 0.2

Este documento corresponde al Proyecto Extra (Proyecto 0.2), el cual amplía la funcionalidad del proyecto original mediante la incorporación de información adicional para el cálculo de cotizaciones.

## Funcionamiento del Programa

El programa fue desarrollado con el propósito de mejorar y aumentar el número de cotizaciones para la empresa TK-U, automatizando el proceso de recolección de datos y cálculo de porcentajes.

El sistema solicita automáticamente los datos personales del cliente, tales como: 

- Nombre
- Edad
- Estado civil
- Cantidad de hijos 


## Problemas que se consideraron 
- Para el desarrollo del programa se tomaron en cuenta los siguientes aspectos:

- La edad del asegurado como factor principal de riesgo.

- El estado civil y la edad del cónyuge.

- La cantidad de hijos declarados.

- La cantidad de propiedades registradas.

- El nivel de ingresos mensuales del solicitante.

- La correcta suma de porcentajes aplicados al precio base.

## Como correr el proyecto
- Copiar el js
- Entrar a la pagina [Jsfiddle](https://jsfiddle.net/)
 - Pegar el js
 - Y presionar el boton Run


## Mejoras por considerar 


- Entre las mejoras que podrían aplicarse al programa se consideran:

- Implementar una interfaz gráfica más amigable para el usuario.

- Validar los datos ingresados para evitar errores.

- Redondear resultados monetarios para mayor claridad.

- Incorporar almacenamiento de cotizaciones en una base de datos.

- Agregar opciones de salida mediante botones.
